# Golden datasets

Reference inputs and outputs generated by the real **pyFAI 2026.5.0** (in the
`daq` conda env) to validate the rsFAI Rust port. See
[`../doc/bit-exact-ladder.md`](../doc/bit-exact-ladder.md) for what each tier
must satisfy.

## Regenerate

```sh
OMP_NUM_THREADS=1 /Users/stevek/mamba/envs/daq/bin/python golden/gen_golden.py
```

`OMP_NUM_THREADS=1` is mandatory: it removes OpenMP reduction nondeterminism so
the accumulation order is reproducible (the bit-exact precondition).

## What's committed vs regenerated

Each `datasets/<config>/` directory is self-contained. To keep the repo lean
(per-pixel arrays are ~100 MB/config), the `.gitignore` commits only:

- `manifest.json` — config + provenance (pyFAI/numpy versions, platform,
  `OMP_NUM_THREADS`, method tuple, per-array dtype/shape, ULP budgets).
- `geometry.poni` — the geometry used.
- `out_*.npy` — the golden **output** curves (radial, intensity, sigma, count,
  sum_signal, sum_variance, sum_normalization[2], std, sem). KB each.

The large per-pixel arrays — `image`, `mask`, `pos0_center`/`pos0_delta`,
`chi_center`/`chi_delta`, `corners`, `preproc`, `solidangle`, `polarization`,
`csr_*` — are **gitignored and regenerated** by `gen_golden.py`. They are needed
for the Tier-A/Tier-B per-pixel tests (M1–M6); run the generator once locally
before those tests. (Future: move them to git-LFS once a remote is configured.)

## Dataset naming

`<detector>__<split>-<algo>-<impl>__<unit>__npt<N>__err<model>`, e.g.
`Pilatus1M__bbox-csr-cython__2th_deg__npt1000__errpoisson`.
